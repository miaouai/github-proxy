# 📝 GitHub Proxy 更新日志

## v2.2 - 2026-03-11

### 🔥 重大修复
**问题**: GitHub 页面内部的所有链接（导航栏、仓库链接、用户头像跳转等）都没有被重写，点击后会直接跳转到 github.com，导致代理失效。

**解决**: 
- 完整重写 HTML 中的 GitHub 链接为 `your-domain.com/gh/github.com/...` 格式
- 支持多种标签属性：`href`, `src`, `action`, `poster`, `data-src`, `data-*`
- 处理 CSS 中的 `url()` 和 `@import`
- 智能处理 `<script>` 标签内嵌的 JSON 数据（GitHub 大量使用这种模式）

**代码示例**:
```javascript
// v2.1: 只替换域名
github.com/user/repo → your-domain.com/user/repo ❌ (会跳转回 GitHub)

// v2.2: 加上 /gh 前缀
github.com/user/repo → your-domain.com/gh/github.com/user/repo ✅ (保持代理)
```

### 🛠️ 技术细节
- 检测是否通过 `/gh` 路径访问（`isProxyPath`）
- 如果是代理路径，HTML 中的所有链接都添加 `/gh` 前缀
- 保留原始 URL 结构，确保重定向正确工作

## v2.1 - 2026-03-11

### 🔒 安全增强
**问题**: 之前的"兼容旧用法"逻辑让所有路径都代理到 GitHub，Cloudflare 可能将整个域名判定为钓鱼网站。

**解决**: 
- 只有 `/gh` 开头的路径才会代理到 GitHub
- 其他所有路径返回 404 友好提示页面

### 🐱 头像加载修复
**问题**: 头像无法加载，HTML 链接重写正则表达式不完善。

**解决**:
- 增强正则匹配，支持完整路径（如 `/u/266430652?v=4`）
- 优化 CORS 和缓存策略
- 同时处理 `style url()` 和 `@import` 等多种格式
